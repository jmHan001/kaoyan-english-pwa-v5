# 拾词：下周逐项排查与修复清单

执行周期：2026-07-27 至 2026-08-02

本轮目标：不增加新功能，集中提高单词内容质量、学习效果、现有功能可靠性和移动端使用体验。

## 范围约束

本轮不修改或扩展：

- 真题模块
- 阅读模块
- 23 点学习验收
- ChatGPT / MCP 自动导入长难句
- 新主题、新页面和其他功能扩展
- 离线状态下的“同步”（离线时只保存到本机）

同步的正确目标是：离线学习正常保存；设备恢复联网后立即尝试同步，并明确显示结果。

## 第一优先级：单词与学习效果

### 7 月 27 日：词库内容质量

- [ ] 确认每日新词从核心词开始，再逐步进入普通词
- [ ] 排查明显错误的中文翻译、词性和熟词僻义
- [ ] 排查重复词、乱码词、残缺词和格式异常词
- [ ] 修正不相关的近义词和关联词
- [ ] 为近义词和关联词显示中文意思，避免只凭熟悉感选择
- [ ] 抽查高考词、考研词和共有词的来源标记
- [ ] 建立词汇问题清单，修改过的词可以追溯

验收：核心词优先正常，抽查词条不存在明显误译、乱关联和显示异常。

### 7 月 28 日：背词判定机制

- [ ] 四个中文释义选项不重复、不泄露答案
- [ ] 错误选项优先从所有已学单词中随机抽取
- [ ] 避免连续多题重复出现相同的三个错误选项
- [ ] 答错或点击“不会”后，连续正确次数立即清零
- [ ] 同一时段连续正确 3 次只算“本轮通过”
- [ ] 已掌握必须经过后续跨日复习验证
- [ ] 右滑返回上一词不重复计分、不改变掌握状态
- [ ] 小测和正式背词使用一致的判定规则

验收：用户不能依靠熟悉的固定错误选项蒙对，掌握率不会被短时间重复答题虚高。

### 7 月 29 日：发音与英文交互

- [ ] 进入单词时自动朗读当前单词
- [ ] 每次答对后立即朗读正确单词
- [ ] 答错后朗读正确答案对应的单词
- [ ] 单词小测接入与背词页一致的语音逻辑
- [ ] 发音加载失败不能阻塞答题或进入下一词
- [ ] 检查单词发音是否完整、清晰，英音和美音按钮是否正确
- [ ] 页面中可交互的英文单词支持点击朗读和查看词义
- [ ] 例句支持完整朗读，避免中途截断
- [ ] 防止快速连续点击造成多段语音重叠

验收：主要背词和小测流程发音及时、完整、不重叠，也不会因为语音接口卡住页面。

## 第二优先级：现有功能可靠性

### 7 月 30 日：学习池与学习记录

- [ ] 今日学习池生成后保持固定，刷新不换词
- [ ] 用户主动选择后才新增、替换或补满单词
- [ ] 自定义每日数量后，主页任务总数同步变化
- [ ] 加背 5、10、20 个词后，任务总数和学习池同步变化
- [ ] 今日任务完成后，刷新、退出和重新打开仍保持完成
- [ ] 同步后今日任务不能从完成状态退回未完成
- [ ] 已背单词、错词、收藏和斩词记录不能互相覆盖
- [ ] 斩词支持恢复，恢复后原来的学习状态仍存在

验收：学习池、任务数量和完成记录在正常操作中始终一致。

### 7 月 31 日：错词、收藏和复习

- [ ] 错词回炉包含全部历史错词
- [ ] 用户可以选择本次随机重温的错词数量
- [ ] 错词重新答错时再次清零连续正确次数
- [ ] 到期复习不会占用每日新词名额
- [ ] 收藏图标在背词卡片右上角正常显示
- [ ] 重点词表在手机端排版正常
- [ ] 收藏、取消收藏后列表立即更新
- [ ] 已背单词列表按时间和状态正确显示
- [ ] 今日完成标准在主页清晰可见

验收：错词、收藏、到期复习和已背单词均能独立工作，不丢记录、不混淆状态。

## 第三优先级：用户体验

### 8 月 1 日：手机端体验统一

- [ ] 今日背词从主页直接进入独立大页面
- [ ] 背完后有明确返回主页入口
- [ ] 页面不再出现重复的背词窗口或入口
- [ ] 检查浅色、珍珠白和暗夜紫主题的文字可见性
- [ ] 修复按钮文字缺失、横线贴字、卡片溢出和收藏排版
- [ ] 禁止双击导致页面意外放大，同时保留正常的可访问性缩放
- [ ] 重要按钮具有点击反馈和加载状态
- [ ] 空学习池、无错词、无收藏等状态提供明确说明
- [ ] 今日任务、剩余词数、已完成状态保持一眼可见
- [ ] 检查 iPhone 安全区、底部导航和横向溢出

验收：手机上所有核心操作单手可完成，页面不拥挤、不跳动、不出现看不见的文字。

## 第四优先级：联网同步稳定性

### 8 月 2 日：同步和最终回归

- [ ] 离线时所有学习记录继续保存在当前设备
- [ ] 恢复联网后立即触发一次同步尝试
- [ ] 同步成功、失败和等待联网均有明确状态
- [ ] 同步失败不能清空或覆盖本地数据
- [ ] 手机和电脑先后修改数据后能够正确合并
- [ ] 检查今日任务、错词、收藏、斩词、长难句和设置的同步
- [ ] 检查跨零点、PWA 更新和旧缓存后的数据状态
- [ ] 运行现有全部单元测试和关键流程测试
- [ ] 发布前分别导出电脑端和手机端备份
- [ ] 分阶段上线，确认手机收到新版本后再完成发布

验收：有网时自动尽快同步，无网时不影响学习；任何同步失败都不会造成数据减少。
